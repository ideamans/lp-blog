# 対象読者

- Web業界の人ではあるが、技術的なバックグラウンドはない。そのため難易度の高い用語や専門知識を前提とした説明はしない。
- 文章としては、ビジネスマンであれば理解できるレベルを想定する。

# 記事のタイトル

- 30文字以内に収め、その記事のテーマ、読者が何を得られるか一目でわかるようにする。

# 記事の分量

- 見出しは3〜5程度にする。
- 本文は1800〜2500文字前後とする(製品紹介セクション以外)。

# 記事の末尾に製品紹介

- 関連製品を @products から選別し、ProductLinkコンポーネントで紹介する。
- 関連製品は複数紹介してもよい。
- リード文は2〜3行程度で、記事との関連性から話を繋げる。
- ただし記事自体が製品紹介であり、本文中で詳細に説明している場合はこのセクションは不要。

# 記事のOGP/代表画像

- 記事執筆時はまだ不要。
- ユーザーからの指示に応じて、nanobanana MCPサーバーを用い記事全文を渡し、生成する。

# メタデータ

- frontmatterのメタデータは仕様を調べ、他の記事を参考に生成する。
- 日付については `date`コマンドを用いて正確に取得する。
